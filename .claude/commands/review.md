# /review — AI Code Review (Local, Pre-Push)

คุณคือ AI reviewer ของทีม ทำตามขั้นตอนนี้ทีละข้อ **ห้ามข้ามขั้นตอนใด**:

## 1. เก็บ diff ที่จะรีวิว

รันคำสั่ง:

\`\`\`bash
git diff main...HEAD
\`\`\`

ถ้า branch หลักของ repo นี้ไม่ใช่ \`main\` (เช่น \`master\`) ให้ตรวจด้วย \`git symbolic-ref refs/remotes/origin/HEAD\` หรือถามผู้ใช้ แล้วใช้ชื่อ branch หลักที่ถูกต้องแทน

ถ้า diff ว่างเปล่า (ไม่มีการเปลี่ยนแปลงเทียบกับ main) ให้แจ้งผู้ใช้ทันทีและหยุดทำงาน — ไม่ต้องสร้างไฟล์รีวิว

## 2. อ่านเกณฑ์กลางของทีม

อ่านไฟล์ \`.github/claude-review-guide.md\` ในโปรเจกต์ ใช้เกณฑ์ในไฟล์นี้เป็นหลักในการตัดสินระดับความรุนแรงของแต่ละประเด็น (🔴 / 🟡 / 🟢) ถ้าไฟล์นี้ไม่พบ ให้แจ้งเตือนผู้ใช้และใช้เกณฑ์มาตรฐานทั่วไปแทนชั่วคราว (parameterized query, auth check, ไม่ hardcode secret, ไม่ log ข้อมูล sensitive เป็นระดับ Critical)

## 3. รีวิวเฉพาะโค้ดที่เปลี่ยนแปลง

รีวิวเฉพาะส่วนที่อยู่ใน diff เท่านั้น (ไม่ต้องไล่รีวิวทั้ง repo) โฟกัสที่โค้ดที่ถูกเพิ่มหรือแก้ไข

## 4. สร้างโฟลเดอร์และเขียนผลรีวิว

สร้างโฟลเดอร์ \`.review/\` ถ้ายังไม่มี (\`mkdir -p .review\`) แล้วเขียนผลรีวิวลงไฟล์ \`.review/review.md\` ด้วยโครงสร้างนี้ **เป๊ะ ๆ**:

\`\`\`markdown
# Code Review

## สรุปการเปลี่ยนแปลง
<สรุปสั้น ๆ ว่า diff นี้ทำอะไร กี่ไฟล์ เปลี่ยนอะไรบ้าง>

## 🔴 Critical (ต้องแก้ก่อน merge)
<รายการประเด็น พร้อมไฟล์:บรรทัด และคำอธิบาย หรือเขียนว่า "ไม่พบประเด็น" ถ้าไม่มี>

## 🟡 Warning (ควรแก้)
<รายการประเด็น พร้อมไฟล์:บรรทัด และคำอธิบาย หรือเขียนว่า "ไม่พบประเด็น" ถ้าไม่มี>

## 🟢 Suggestion (แก้หรือไม่ก็ได้)
<รายการประเด็น พร้อมไฟล์:บรรทัด และคำอธิบาย หรือเขียนว่า "ไม่มีข้อเสนอแนะเพิ่มเติม" ถ้าไม่มี>

REVIEWED-COMMIT: <ผลลัพธ์จากคำสั่ง git rev-parse HEAD>
\`\`\`

**สำคัญ:** บรรทัดสุดท้ายของไฟล์ต้องเป็น \`REVIEWED-COMMIT: <hash>\` เท่านั้น (ไม่มีข้อความอื่นต่อท้าย) โดย \`<hash>\` ต้องมาจากการรันคำสั่ง \`git rev-parse HEAD\` จริง ๆ ณ ตอนนี้ ไม่ใช่ค่าที่เดาหรือจำจากก่อนหน้า

## 5. Copy ผลรีวิวลง clipboard

ตรวจ OS ก่อนแล้วเลือกคำสั่งให้เหมาะสม:

\`\`\`bash
# macOS
cat .review/review.md | pbcopy

# Windows (Git Bash / WSL ที่เรียก clip.exe ได้)
cat .review/review.md | clip.exe

# Linux (ต้องมี xclip ติดตั้งไว้)
cat .review/review.md | xclip -selection clipboard
\`\`\`

ตรวจสอบ \`uname -s\` หรือ environment เพื่อเลือกคำสั่งที่ถูกต้องอัตโนมัติ ถ้า copy ไม่สำเร็จ (เช่นไม่มี xclip/clip.exe) ให้แจ้งผู้ใช้ตรง ๆ ว่า copy ไม่สำเร็จ พร้อมบอกให้เปิดไฟล์ \`.review/review.md\` แล้ว copy เอง — **ห้ามเงียบแล้วข้ามไป**

## 6. แจ้งผลกับ developer

หลังจากเขียนไฟล์และ copy เสร็จ ให้สรุปให้ developer ทราบ (สั้น กระชับ):

- จำนวนประเด็นแต่ละระดับ (🔴/🟡/🟢) ที่พบ
- ถ้ามี 🔴 ให้เตือนว่าควรแก้ก่อน push
- ผลรีวิวถูก copy ลง clipboard แล้ว ให้นำไปวางเป็นคอมเมนต์ใน PR บน GitHub หลัง push และเปิด PR เรียบร้อย
- เตือนว่าถ้ามีการ commit เพิ่มหลังจากนี้ ต้องรัน \`/review\` ใหม่ เพราะ hash จะไม่ตรงกับ commit ล่าสุดอีกต่อไป
